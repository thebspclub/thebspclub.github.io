    multisource
    Half-Life misc entity
    description

    The multisource entity acts as an AND gate. That is, it allows an event to be activated only if a certain condition has been met.

    usage

    If an entity such as a door or trigger specifies a multisource entity as its master, the master must be triggered before the door or trigger will work.

    properties

  • targetname (target_source) Name - A name must be given to the multisource so other entities can use it as a master.
  • target (target_destination) Target - (see below)
  • globalstate (string) Global State Master - This is the name of the global variable (set by an env_global) that can be used to control the state of the multisource.

    • notes

    Target Usage
    The Target (target) property of the multisource allows you to use the multisource as a special case trigger_relay. Let's use the example below -

      You've got a sound that you want to be played when four buttons are in their pushed state. If any of the buttons return to their "un-pushed" state before the last is pressed, the sound will not play.
    Using the multisource, this is simple. Make the buttons all target a single multisource entity. Make the multisource target the ambient_generic sound. Now, whenever all of the multisource's "children" are active, its target will get activated.
