    func_tracktrain
    Half-Life func entity
    description

    The func_tracktrain entity creates a moving controllable platform capable of following a path.

    usage

    When you create the brush model, it must be facing west. When it spawns in the game, it will automatically move to the first path_track and orient itself to face the second path_track. The first thing you should do after creating the entity is give it a name, as most entities that interact with the tracktrain will be required to reference it.

      Note: the func_tracktrain is one entity that can have CLIP brushes as part of itself. (A CLIP brush is a brush that is completely covered with the CLIP texture. It blocks the player's movement but allows ammo to pass through it) You can use CLIP brushes to block player movement -- for example to stop him from exiting the train.
    An important part of the func_tracktrain creation is the origin brush. The origin brush location determines how the func_tracktrain will "attach" to the path_track path points. To add the origin brush to the func_tracktrain, just make a brush with the 'ORIGIN' texture on all sides, and make it part of the entity. The origin brush will not be visible or tangible in the game, and is only used for path_track guidance.

    Another important part of the func_tracktrain is the func_traincontrols. This lets the player control the speed and direction of the train. The controls are not part of the tracktrain entity itself, rather the func_traincontrols entity is placed in the area that you want the controls to be, and that areas stays defined with the train when it is in the game.

    properties

  • targetname (target_source) Name - A number of entities that deal with the track train require it to have a name as a reference point.
  • target (target_destination) First stop target - This has to point to the Name (targetname) of a path_track entity. The train will start positioned at this path point, aimed at the next path point.
  • bank (string) Bank angle on turns (default: 0) - If you want the train to bank (tilt) when it goes around a corner, set the angle here.
  • dmg (integer) Damage on crush (default: 0) - If a player is caught between the train and a hard place, he'll take this much damage.
  • height (integer) Height above track (default: 4) - This is the height above the path_tracks that the train will ride, based on the location of the train's origin brush.
  • speed (integer) Speed (units per second) (default: 64) - If the train is user controllable, this is the maximum speed the user can make the train go.
  • startspeed (integer) Initial speed (default: 0) - This is the speed the train starts at. If the train is not user controllable, this is the only speed the train will run at unless set through a path_track.
  • wheels (integer) Distance between the wheels (default: 50) - This determines how the train turns around corners. Generally, the length of the train will work fine here.
  • sounds (integer) Sound
  • volume (integer) Volume (10 = loudest) (default: 10)
  • globalname (string) Global Entity Name
  • Rendering Properties
  • ZHL Properties

    • Flags
    This entity has the following flags. Flags can be either on or off. In the editor, these flags can be accessed in the Flags section of the entity properties.

  • 1 - No Pitch (X-rot) - If this flag is set, the train will only rotate on the Z axis (ie: turning left or right).
  • 2 - No User Control - Set this flag to ensure that the player cannot control the train's movement.
  • 8 - Passable - If this flag is set, the train will not be solid.

    Mon Feb 02, 2004 / 03:17pm PST
      Just something I've noticed with this entity, regarding the lighting (specifically the HL subway car)

    HL seems to render the lighting of the train entity using the lights that shine on it in its starting position i.e. when the train is rotated facing west.

    For example, if you had lighting outside the train, but one of those light sources was on or in the train in its rotated starting position (before it is aligned to the track) that beam of light will be rendered on it. For me, this gave the inside of the train the cool yellow glow of the station mixed with the white of the train lights (when it only should have had the train lights).

    Having the train entity start inside a hollowed box (textured with "black") will render the lighting as it should be inside the train, and then the entity will jump to the correct starting point when the map loads.
