    path_corner
    Half-Life misc entity
    description

    You can use the path_corner entity to create a linked string of path points for something like a func_train to follow.

    usage

    Simply give each path_corner a Name (targetname) and Target the next path_corner in the path.

    You can control whether the train stops at this point for a number of seconds (specified as the Wait value) and give the train a new speed which will take effect once the train passes over the path_corner.

    The toggle spawnflag lets you make the train stop on the current path corner and wait to be triggered again before moving. This is useful for lifts and moving platforms that the player can trigger. The teleport spawnflags makes a func_train teleport to the current path_corner.

    properties

  • targetname (target_source) Name
  • target (target_destination) Next stop target
  • speed (integer) New Train Speed (default: 0)
  • wait (integer) Wait here (secs) (default: 0)
  • message (target_destination) Fire On Pass
  • angles (string) Pitch Yaw Roll (Y Z X) (default: 0 0 0)

    • Flags
    This entity has the following flags. Flags can be either on or off. In the editor, these flags can be accessed in the Flags section of the entity properties.

  • 1 - Wait for retrigger
  • 2 - Teleport
  • 4 - Fire once

    Brian 'Yamazaki' Dorner
    Thu Apr 10, 2003 / 06:13pm PDT
      There is a little known bug with how the Fire On Pass field works for path_corner. Fire On Pass will not properly trigger entities that share the same name. For example, if a func_train arrives at a path_corner, and that path_corner's Fire On Pass targets 'bob', and more than one entity bears the name 'bob', then you might see odd behaviour.

    I've confirmed this behaviour when a Fire On Pass tried to target four func_trains that had the same name. The func_trains made noise as if they were moving, but didn't budge an inch, and the sound looped forever. Remove all but one of the func_trains and it will work.

    The solution is to use a trigger_relay, multimanager, or other triggering entity, between the path_corner and the multiple entities.
