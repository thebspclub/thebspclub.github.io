    func_tank
    Half-Life func entity
    description

    Use this to create a controlable gun turret.

    usage / properties

    The gun should be created facing angle 0 (in the top view, it should point to the right). Use the angle indicator to specify which direction it should face in the game.

    1. The first step is, of course, to build the brush model of the gun. It can be anything you like, although normally you'd want it to look like some type of gun. It is important to note that an origin brush must be included in the construction of the gun. An origin brush is a brush covered in the ORIGIN texture. Any entity that rotates absolutely *must* have an origin brush as part of their makeup.

    2. The func_tank brush model must be built facing east (to the right in the top view). Later, when you turn it into a func_tank, you will set the actual facing of the gun in the game by setting the angle value of the entity.

    3. Turn the brush model into a func_tank. Select it, and setup its properties.

    4. targetname (target_source) Name - The gun must have a unique name. This will be used to control the gun through the func_tankcontrols (see below).
    5. target (target_destination) Target - The Name (targetname) of an ambient_generic should be specified here. This will be the sound the gun makes when fired.
    6. bullet (integer) Bullets - Set the bullet type here.

           0 - None (default)
           1 - 9mm
           2 - MP5
           3 - 12mm

    7. barrel (string) Barrel Length (default: 0) - This is the length from the center of the origin brush to the tip of the barrel.
    8. barrely (string) Barrel Horizontal (default: 0) - This is the horizontal distance from the center of the origin brush to the center of the barrel.
    9. barrelz (string) Barrel Vertical (default: 0) - This is the vertical distance from the center of the origin brush to the cetner of the barrel.
    10. bullet_damage (string) Damage Per Bullet (default: 0) - Set the bullet damage here.
    11. firerate (string) Rate of Fire (default: 1) - This is the rate of fire per second.
    12. master (string) (Team) Master
    13. maxRange (string) Maximum target range (default: 0)
    14. minRange (string) Minmum target range (default: 0)
    15. persistence (string) Firing persistence (default: 1) - This is the firing persistance of the gun. This is really only useful for non-user controlled turrets (see below). This will make the gun continue to fire for a moment after the player leaves its visibility range, rather than being instant on/off.
    16. pitchrange (string) Pitch range (default: 0) - This is the range of up/down movement in degrees.
    17. pitchrate (string) Pitch rate (default: 0) - This is the speed of the up/down movement of the gun. For a fast feeling, a value of 600 is good.
    18. pitchtolerance (string) Pitch tolerance (default: 5)
    19. rotatesound (sound) Rotate Sound - Specify the targetname of an ambient_generic here.
    20. spriteflash (string) Flash Sprite - This is the sprite that will be appended at the end of the barrel when the gun is fired, sprites/muzzleflash2.spr for example.
    21. spritescale (string) Sprite scale (default: 1) - Use this to set the scale of the sprite, if the default size is not to your liking.
    22. spritesmoke (string) Smoke Sprite
    23. yawrange (string) Yaw range (default: 180) - This is the range of left/right movement in degrees.
    24. yawrate (string) Yaw rate (default: 30) - This is the speed of the left/right movement of the gun. For a fast feeling, a value of 600 is good.
    25. yawtolerance (string) Yaw tolerance (default: 15)
    26. firespread (integer) Bullet accuracy - Set the bullet accuracy here.

           0 - Perfect Shot (default)
           1 - Small cone
           2 - Medium cone
           3 - Large cone
           4 - Extra-large cone

    27. ZHLT Properties
    28. Rendering Properties
    29. globalname (string) Global Entity Name
    30. angles (string) Pitch Yaw Roll (Y Z X) (default: 0 0 0)

      31. This entity has the following flags. Flags can be either on or off. In the editor, these flags can be accessed in the Flags section of the entity properties.

    31. 1 - Active - this will make the gun start active. It is useful for making monster controlled turrets. This is unneeded for user controlled turrets.
    32. 16 Only Direct - If this gun is set to fire automatically (instead of controlled by the user), setting this flag will make the gun only fire if the target is in direct line of sight.
    33. 32 - Controllable - If this turret is going to be controllable by the user, this flag must be checked.

    34. Create a func_tankcontrols entity just behind the func_tank. Set it to target the Name (targetname) of the func_tank.

    35. Place a clip brush over the func_tank so the player can't accidentally interfere with its movement. This may require a bit of tweaking until it is just right. A clip brush is a brush covered in the CLIP texture. It is invisible in the game, but blocks player movement. Ammo *will* pass through a clip brush, however.

    36. If you haven't created the ambient_generic that is targeted by the func_tank yet, now is the time to do it. A good sound to use for a gun is the ambience/biggun1.wav sound. In the flags section of the ambient_generic properties, set its radius, and make it 'Start Silent' and 'Not Looped'.


    Monster Controlled Turrets
    In various places in Half-Life, you will have seen gun turrets that are being controlled by military grunts. These are quite simple to create once you have the standard func_tank created.

    1. Create the func_tank, as above.

    2. Select the func_tank and press Alt+Enter to bring up its properties. Click on the Flags section. The only flags you want checked are 'Active' and 'Player'.

    3. A monster controlled func_tank does not require a func_tankcontrols entity, so if one is present (and connected to this func_tank) remove it.

      The above are the only diffences in the func_tank, between a normal user controlled func_tank, and a monster controlled version.

    4. Now, place the controller. In this example, we'll use a monster_human_grunt. Place it in a position just behind the gun.

    5. Select the monster and press Alt+Enter to bring up its properties. In the TriggerTarget field, enter the Name (targetname) of the func_tank. In the Trigger Condition field, select "Death".

    6. In the monster's Flags properties, place a check beside the Prisoner flag. This will make the monster stay by the gun (making it look like he is controlling it). Otherwise, the monster will run at you and the gun will continue to fire.


    When this is placed in the level, the grunt (well, the turret) will fire at the player whenever he has site of him. Killing the grunt will disable the turret.
