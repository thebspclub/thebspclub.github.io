    env_sprite
    Half-Life env entity
    description

    The env_sprite entity lets you place sprites in your level.

    properties

  • targetname (target_source) Name - If you give the env_sprite a name, triggering it will turn it on and off. You can use the flags (see below) to control its triggered behavior.
  • framerate (string) Framerate (default: 10.0) - If the sprite is animated (sprites/steam1.spr, for example) this value controls how many frames per second the animation will run at.
  • model (sprite) Sprite Name (default: sprites/glow01.spr)- this is the path and name of the sprite as it is in the pak file. The value here should always be sprites/spritename.spr. Note the forward slash (/). It will not function correctly if you use a normal backslash ().

      Note: A complete list of sprites can be found in the pak0.pak file which is located in your half-lifevalve direcory. You can browse the PAK file with a PAK browser (like the Half-Life Model Viewer or Wally). You can view the actual sprite file with SprView.
  • scale (string) Scale - you can adjust the size of the sprite by defining its scale here. Default is 1:1 scale. (Render Mode should be set to Additive if you want to change the scale)
  • Rendering Properties

      Note: To get rid of a black box area around your sprite, change the Render Mode to Additive or Glow.
  • angles (string) Pitch Yaw Roll (Y Z X) (default: 0 0 0)

    • Flags
    This entity has the following flags. Flags can be either on or off. In the editor, these flags can be accessed in the Flags section of the entity properties.

  • 1 - Start on - If the env_sprite has a name, it will need to be triggered to appear. If this flag is set, it will start visible. Triggering it will cause it to disappear.
  • 2 - Play Once - In the case of an animated sprite, the sprite will go through its animation only once when this flag is checked. Each time it is triggered, it will play once and wait to be triggered again.
