    func_mortar_field
    Half-Life func entity
    description

    The func_mortar_field entity is used to define an area from which air strikes can be called.

    usage

    The entity is extremely simple to configure.

    First, of course, give it a Name (targetname). This is needed in order to trigger it (with a button or some other trigger). Give it a spread radius (m_flSpread). This defines the accuracy of the air strike bombs. The repeat count (m_iCount) determines how many bombs will be dropped per air strike. The targeting value (m_fControl, 0 - random, 1 - activator, or 2 - table) should be set to 2 (table) if you are using a table to define the air strike location (as in the example map). The X and Y controllers (m_iszXController and m_iszYController) coincide with the targetnames of the momentary_rot_buttons that define the hit location on the table.

    The example map is quite simple and should be quite helpful if you disect it.

    properties

  • m_flSpread (integer) Spread Radius (default: 64)
  • m_iCount (integer) Repeat Count (default: 1)
  • m_iszXController (target_destination) X Controller
  • m_iszYController (target_destination) Y Controller
  • m_fControl (integer) Targeting

         0 - Random (default)
         1 - Activator
         2 - Table

  • targetname (target_source) Name
