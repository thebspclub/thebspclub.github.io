    trigger_relay
    Half-Life trigger entity
    description

    The trigger_relay acts as a relay (surprise!) between a trigger and an action.

    usage

    This is most useful when you want to start a number of events with one trigger. Simply create a number of trigger_relays, each with the same targetname, and set the targets and delays of the individual trigger_relays as you like. When you want to trigger this sequence of events, use another trigger to target the name of the trigger_relay group.

      Note: although the above is the purpose of a trigger_relay, its job has been made obsolete by the multi_manager entity which is much more of an elegant solution to setting up a sequence of events.
    properties

  • targetname (target_source) Name - The relay must be activated by another event.
  • target (target_destination) Target - This is the event that will be activated.
  • killtarget (target_destination) KillTarget
  • delay (string) Delay before trigger (default: 0)
  • triggerstate (integer) Trigger State - If it is set to toggle, it will act as a normal trigger. If set to on, it will only affect an event that is currently off or inactive. If set to off, it will only affect an event that is currently on or active.

         0 - Off (default)
         1 - On
         2 - Toggle

    • Flags
    This entity has the following flags. Flags can be either on or off. In the editor, these flags can be accessed in the Flags section of the entity properties.

  • 1 - Remove On fire

    • notes

    Jay Stelly adds this note:

      "Normal triggers are always toggle. This creates a bunch of problems that the trigger_relay can solve. Since objects can have delays, normal triggering can result in race conditions where the state of the triggering object and the state of the triggered object don't match up. If the messages are explicit "Turn object ON", "Turn Object Off", then even when the state disagrees, it will fix itself on the next message.

      For instance: You've got a button that turns on a light and another button to turn it off. You can't do that without trigger_relay. One button sends through a relay that sends an "ON" message and the other goes through a relay to send an "OFF" message. "ON" always turns on the door, (if it's already on it does nothing) "OFF" always turns it off. The definition of "ON" and "OFF" depend on how you set up your object, but the same basic principle applies to switchable lights, func_trains, etc."
    The above example is demonstrated in the relay1 example map (below).
